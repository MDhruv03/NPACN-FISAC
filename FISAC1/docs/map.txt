Question - 
Group-12	Design and implement a secure real-time Location Sharing Web Application where authenticated users can broadcast their live geographic coordinates to a central server, and the server distributes updates instantly to authorized connected clients using WebSockets over TCP. The system must demonstrate correct TCP socket programming, concurrent client handling, persistent database storage of location updates and session logs, secure communication practices, advanced socket option configuration, and reliable connection management.
•	Analyze the complete TCP and WebSocket communication workflow of the system, including TLS-secured connection establishment, WebSocket handshake, full-duplex communication, and graceful termination, and justify the concurrency model used in the server to maintain data integrity and scalability.
•	Implement a concurrent server that authenticates users, receives continuous location updates, stores them in a database with proper timestamping, logs connection events (syslogs), and efficiently broadcasts updates to authorized subscribers while correctly handling partial transmissions and network latency.
•	Configure and experimentally evaluate socket options such as SO_REUSEADDR, SO_KEEPALIVE, SO_RCVBUF, and TCP_NODELAY, and analyze their impact on latency, throughput, secure session persistence, and database write performance under continuous update conditions.
•	Evaluate system robustness under abnormal conditions such as sudden client disconnection, high update frequency, network congestion, and unauthorized access attempts, and analyze relevant TCP states including TIME_WAIT and CLOSE_WAIT, along with logging accuracy and database consistency.
